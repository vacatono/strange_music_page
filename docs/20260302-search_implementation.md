# 検索機能の導入と Fuse.js への移行 (2026-03-02)

## 概要

サイト内検索機能として当初 Pagefind を導入しましたが、日本語の固有名詞（人名・アーティスト名など）の検索精度を向上させるため、**Fuse.js** を使用したクライアントサイド検索に移行しました。

## Fuse.js への移行理由

- **単語分割問題の解消**: Pagefindは日本語を自動的に分割（トークナイズ）するため、「渡辺等」が「渡辺」と「等」に分かれてしまい、正確な部分一致が困難でした。
- **自由な部分一致検索**: Fuse.jsは文字列のサブストリング一致に強く、日本語のアーティスト名や曲名を直感的に検索できます。
- **リアルタイム更新**: `npm run dev` 中でも、Markdownファイルを編集すると即座に検索インデックス（`search-index.json`）に反映されます。

## 実装の詳細

### 1. インデックス生成 (`src/pages/search-index.json.ts`)

Astroのエンドポイントを作成し、全レポートから以下の情報を抽出したJSONを生成するようにしました。

- タイトル、日付、アーティスト、会場
- 本文の冒頭（スニペット用）

### 2. 検索UIコンポーネント (`src/components/SearchUI.tsx`)

React と Fuse.js を使用したカスタム検索UIを実装しました。

- **インクリメンタルサーチ**: 1文字入力するごとに結果が絞り込まれます。
- **プレミアムなデザイン**: サイトのテーマカラーに合わせた、清潔感のあるカード型レイアウト。

### 3. 使用方法

トップページの検索窓に文字を入力するだけで、即座に全レポートから該当するものをリストアップします。

## 動作確認モード

Pagefindとは異なり、以下のどちらのモードでも完全に動作します。

- `npm run dev` (開発モード)
- `npm run build` -> `npm run preview` (本番確認モード)

## 今後の拡張性

- **ソート機能**: 日付順での並び替えなどの追加が容易です。
- **詳細フィルタ**: 会場や年などのプロパティによる絞り込みの強化が可能です。
